# Design Context — Browserbase Navigator

## Target audience
Developers evaluating or building on the Browserbase ecosystem (Stagehand, browse CLI,
MCP server, Search/Fetch APIs, Agents, Functions) — plus the Browserbase team itself
as a hiring audience. They live in terminals and docs sites; they smell templated UI
instantly.

## Use cases
- "Which Browserbase tool fits my job?" — decision questions answered with citations
- Exact-identifier lookups (keepAlive, REQUEST_RELEASE) and ecosystem-wide Q&A
- "What changed recently?" across the whole surface (git-diff-backed)
- Save a recommendation as an immutable, shareable plan permalink

## Brand personality / tone
Browserbase-native, calibrated live against browserbase.com + docs.browserbase.com
(2026-07-07, user-approved). Authoritative source: `../../docs/browserbase-design-dna.md`.
- One loud accent: OrangeRed #FF4500. Everything else warm stone neutrals — never zinc.
- Dark default: near-black flat canvas, cards barely lighter, hairline borders carry
  ALL structure (no drop shadows, no border soup — user: "less borders").
- Light theme: WHITE canvas (not cream — user override of the DNA doc), warm tints
  only in surface-2/sunken.
- Mono uppercase eyebrows carry the terminal texture; weight-500 display headlines
  with tight negative tracking; declarative terse microcopy ("Ask the ecosystem.").
- Signature surfaces: operator step-rail (flat mono-badged tool rows), macOS
  traffic-light chrome on browser previews, orange hairline hover on cards.
- No emoji. lucide line icons. One easing: cubic-bezier(0.3,0,0.15,1).
- Reference peers: docs.browserbase.com Assistant panel, director.ai (IA), morphic
  (answer-engine UX). Differentiator to protect: citations + tool transparency —
  the visible work IS the brand.

## Constraints
- Next.js 16 + Tailwind v4 (CSS-first, no config file) + shadcn/ai-elements.
- Token-only colors (globals.css :root dark / .light) — hardcoded hex only in the
  logo mark and the plan-page print block. AA verified pairings per the DNA doc.
- prefers-reduced-motion gates everything, incl. Framer (useReducedMotion — the
  global CSS rule does not govern it).
- Build spec + file ownership map: `docs/redesign-spec.md`.
